## 1. 产品定位

Snare Practice Lab 是一款面向个人练习的架子鼓节奏训练系统。

它不是教学管理软件，也不是单纯节拍器，而是一个围绕个人练习闭环设计的工具：

```text
选择节奏型
→ 编排练习
→ 跟练播放
→ 调整 BPM
→ 记录练习
→ 沉淀目标和笔记
→ 下次继续
```

第一阶段以军鼓节奏训练为核心，但底层节奏型设计不绑定军鼓，未来可以复用于套鼓 groove、真实歌曲片段和 MIDI 评分。

## 2. 产品目标

### 2.1 MVP 目标

MVP 阶段要实现一个用户每天可以真正打开使用的个人练习系统。

核心目标包括：

1. 能快速选择基础节奏型。
2. 能随机或顺序生成练习。
3. 能以五线谱 / 节奏谱形式进行跟练。
4. 能在跟练页调整 BPM。
5. 能记录个人目标、练习笔记、问题记录和练习回顾。
6. 能保留当前统一的深色练习驾驶舱设计风格。
7. 能在桌面、平板和小屏幕上保持响应式布局。

### 2.2 非目标

MVP 阶段暂不做：

- 手机原生 App
- 云同步
- 账号系统
- 老师端 / 教学后台
- AI 鼓谱转写
- 完整套鼓谱编辑
- 麦克风识别
- MIDI 准确度评分
- 游戏化排行榜
- 商业曲库

## 3. 核心用户场景

### 3.1 在家使用电子鼓练习

用户在家用电子鼓或 MIDI 设备练习。

需要：

- 打开首页查看今天练什么
- 继续上次练习
- 选择或生成节奏型
- 进入跟练页
- 调整 BPM
- 保存练习记录
- 记录问题和感受

### 3.2 使用哑鼓垫练习

用户偶尔使用哑鼓垫。

需要：

- 看五线谱 / 节奏谱
- 听节拍器
- 练习手法、重音、读谱
- 手动记录练习结果

### 3.3 自学或上课后的个人整理

用户可能来自：

- 线下上课
- 自学视频
- 教材练习
- 自己发现的问题
- 临时灵感

软件不假设用户一定上课，因此统一使用：

- 个人目标
- 练习笔记
- 问题记录
- 练习回顾

## 4. 设计原则

### 4.1 视觉风格

必须保持当前 V3 / V4 / V6 原型中的设计风格：

- 深色背景
- 卡片式模块
- 冷色信息层级
- 黄色作为核心强调色
- 红色作为播放线、重点状态、警示状态
- 绿色用于完成、进度、积极反馈
- 整体感觉是“练习驾驶舱”，不是教学平台，也不是纸面练习册

### 4.2 响应式布局

产品采用 Responsive Web 设计，而不是单独设计手机端。

| 屏幕类型 | 布局策略 |
|---|---|
| 桌面大屏 | 左侧完整导航 + 主内容多栏 |
| 平板横屏 | 左侧导航 + 主区域双栏 |
| 平板竖屏 | 主内容单栏，设置模块上移 |
| 小屏幕 | 底部导航 + 卡片纵向堆叠 |
| 微信小程序 | 后续基于同一信息架构重做小程序布局 |

视觉风格不随屏幕改变，只改变排布和信息密度。

## 6. 功能需求

## 6.1 首页 Home

### 6.1.1 模块定位

首页是个人练习驾驶舱，用于回答：

> 今天该练什么？

首页不是节奏生成器，也不是日志列表，而是把当前练习状态、个人目标和下一步建议集中呈现。

### 6.1.2 功能项

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| H-001 | 继续上次练习 | P0 | 显示上次练习名称、BPM、时长、目标重复次数 |
| H-002 | 开始跟练入口 | P0 | 从首页直接进入跟练页 |
| H-003 | 调整节奏入口 | P0 | 进入节奏型选择与练习编排 |
| H-004 | 新建练习笔记 | P0 | 记录练习想法、问题观察、下一步目标 |
| H-005 | 当前个人目标 | P0 | 展示正在进行的目标 |
| H-006 | 最近练习 | P0 | 展示最近练习记录 |
| H-007 | 本周进度 | P1 | 练习时长、目标完成度、稳定 BPM |
| H-008 | 当前 Focus | P1 | 聚合当前重点，如十六分、重音、Paradiddle |
| H-009 | 下一步建议 | P1 | 根据目标和练习记录给出建议 |
| H-010 | 快速入口 | P0 | 随机读谱、顺序练习、新建笔记 |

### 6.1.3 首页用词

统一使用：

- 当前个人目标
- 练习笔记
- 问题记录
- 练习回顾
- 下一步建议

禁止使用：

- 课堂作业
- 上课记录

## 6.2 节奏型选择 Rhythm Pattern Selection

### 6.2.1 模块定位

节奏型选择是产品的核心记忆点。

用户应该通过音符卡片选择节奏，而不是只看到抽象网格。

### 6.2.2 显示方式

每个节奏型卡片应包含：

| 元素 | 说明 |
|---|---|
| 音符可视化 | 用音符表达节奏型 |
| 节奏名称 | 如四分、十六分、切分、后半拍 |
| 击打数量 | 如 2/4、3/4 |
| 圆点辅助 | 表示一拍内的击打位置 |
| 选中状态 | 黄色描边或黄色高亮 |
| 收藏状态 | 后续支持 |

### 6.2.3 功能项

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| R-001 | 显示十六分节奏型 | P0 | 一拍内 4 个位置的组合 |
| R-002 | 显示三连音节奏型 | P0 | 一拍内 3 个位置的组合 |
| R-003 | 音符卡片可视化 | P0 | 用户优先看到音符 |
| R-004 | 圆点辅助显示 | P0 | 帮助理解击打位置 |
| R-005 | 多选节奏型 | P0 | 控制随机或顺序练习范围 |
| R-006 | 我的收藏 | P1 | 收藏常用节奏型 |
| R-007 | 难度标签 | P1 | 基础、进阶、困难 |
| R-008 | 搜索 / 筛选 | P2 | 后续扩展 |

## 6.3 练习队列 Practice Queue

### 6.3.1 模块定位

练习队列用于将多个节奏型组织成一次练习。

支持两种核心训练方式：

1. 随机练习：训练读谱反应。
2. 顺序练习：训练固定组合和个人目标。

### 6.3.2 练习模式

| 模式 | 说明 | 优先级 |
|---|---|---|
| 随机打乱 | 根据已选节奏型随机生成练习 | P0 |
| 按顺序练习 | 按用户排列的顺序播放 | P0 |
| 每轮重洗 | 每一轮重新随机排序 | P1 |
| 无尽模式 | 持续生成后续小节 | P1 |

### 6.3.3 功能项

| 编号 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| Q-001 | 添加节奏型到队列 | P0 | 从选择区加入 |
| Q-002 | 队列预览 | P0 | 显示练习顺序 |
| Q-003 | 随机打乱 | P0 | 生成随机顺序 |
| Q-004 | 按顺序练习 | P0 | 保持用户排列 |
| Q-005 | 拖拽排序 | P1 | 用户手动调整顺序 |
| Q-006 | 保存队列 | P1 | 保存为个人目标或练习 |
| Q-007 | 每轮重洗 | P1 | 每轮随机重排 |
| Q-008 | 无尽模式 | P1 | 长时间随机读谱 |

## 6.4 跟练 Trainer

### 6.4.1 模块定位

跟练页面是实际练习发生的地方。

跟练页应优先显示五线谱 / 节奏谱，而不是网格。

### 6.4.2 页面组成

| 区域 | 内容 |
|---|---|
| 谱面区域 | 五线谱 / 节奏谱 |
| 拍点区域 | 当前拍点圆点 |
| 播放线 | 红色播放线 |
| 播放控制 | 开始、暂停、继续、重置 |
| BPM 控制 | 拨盘、+1、-1、+5、-5、自动升速 |
| 声音层 | 节拍器、鼓点、重音、无尽模式 |
| 编辑入口 | 长按音符局部编辑 |
| 保存入口 | 练习结束后保存记录 |

### 6.4.3 BPM 需求

BPM 的主控位置在跟练页。

| 功能 | 优先级 |
|---|---|
| 显示当前 BPM | P0 |
| 拨盘式调节 | P0 |
| +1 / -1 微调 | P0 |
| +5 / -5 快速调整 | P1 |
| 自动升速 | P1 |
| 保存本次实际 BPM | P0 |

### 6.4.4 长按编辑

跟练页支持长按音符进行局部编辑。

长按后弹出编辑浮层，支持：

| 操作 | 优先级 |
|---|---|
| 改为休止 | P1 |
| 添加 / 删除重音 | P1 |
| 设置 R / L 手法 | P1 |
| 标记 ghost note | P2 |
| 删除音符 | P1 |
| 拆分节奏 | P2 |

## 7. 数据模型草案

  updatedAt: string

type PracticeQueue = {

type PracticeSession = {

- 用户可选择随机或顺序
- 生成结果可进入跟练页

- 可以调整 BPM

## Phase 6：局部编辑与自定义增强

目标：提高练习编辑效率。

任务：

1. 长按音符弹出局部编辑浮层。
2. 改为休止。
3. 添加 / 删除重音。
4. 设置 R / L。
5. 标记 ghost note。
6. 删除音符。
7. 编辑后更新练习队列。
8. 保存为自定义练习。

## 10. MVP 范围建议

建议第一个可用版本只做到 Phase 0 到 Phase 4 的核心闭环：

```text
首页
→ 节奏型选择
→ 练习队列
→ 跟练播放
→ BPM 调整
→ 保存练习记录
```

MVP 必须包含：

- 深色练习驾驶舱 UI
- 响应式布局
- 首页
- 节奏型音符卡片
- 随机 / 顺序练习
- 跟练五线谱
- BPM 拨盘
- 节拍器
- 鼓点示范
- 练习记录
- 个人目标
- 练习笔记

MVP 可以暂缓：

- MIDI 评分
- 长按复杂编辑
- 套鼓谱
- AI 转写
- 云同步
- 微信小程序

## 11. 当前最终产品一句话

Snare Practice Lab 是一个深色驾驶舱风格的个人架子鼓练习系统，帮助用户选择节奏型、生成随机或顺序练习、看五线谱跟练、调整 BPM、记录个人目标与练习笔记，并逐步扩展到 MIDI 准确度评分和套鼓练习。
