# SnareLab 第一阶段 UI 规范说明书

版本：Phase 1 / v1  
视觉基准：[snarelab-phase1-pwa-v6-refined-blue-minimal.png](/Users/cheche/Documents/SnareLab/docs/prototypes/snarelab-phase1-pwa-v6-refined-blue-minimal.png)  
适用范围：PWA 第一阶段页面，包括「今天」「专注计时」「保存本次练习」「分类」「统计」。

## 1. 产品与设计原则

SnareLab 第一阶段是一个架子鼓练习计时与统计工具，不做编谱、不做复杂练习库。当前 UI 目标是：快速开始练习、低干扰计时、练后归类、轻量查看统计。

设计关键词：

- 简约、清爽、轻量
- 白色 / 浅灰为主，蓝紫色作为唯一主识别色
- 功能优先，避免装饰性过强
- 鼓练习气质通过节奏纹理、圆环刻度、计时数字体现，不使用拟物鼓图
- 移动端优先，PWA 安装后接近原生 App 使用体验

第一阶段底部导航只保留 3 个主 Tab：

```text
今天 / 分类 / 统计
```

不得出现独立「记录」Tab。今日记录合并在「今天」页。

## 2. 色彩规范

### 2.1 基础颜色

| Token | 色值 | 用途 |
|---|---:|---|
| `--color-bg` | `#F7F8FA` | App 全局背景 |
| `--color-surface` | `#FFFFFF` | 卡片、底部弹窗、导航容器 |
| `--color-text` | `#171717` | 主文本、标题、重要数字 |
| `--color-text-secondary` | `#8E8E93` | 辅助说明、日期、次要标签 |
| `--color-text-tertiary` | `#B8B8BE` | 禁用态、弱提示 |
| `--color-border` | `#ECEEF2` | 卡片描边、输入框边框、分隔线 |
| `--color-shadow` | `rgba(15, 23, 42, 0.06)` | 轻阴影 |

### 2.2 品牌主色

| Token | 色值 | 用途 |
|---|---:|---|
| `--color-primary` | `#5B63F6` | 主按钮、选中态、计时进度、当前 Tab |
| `--color-primary-hover` | `#4E56E8` | Web hover / pressed 加深 |
| `--color-primary-soft` | `#EEF0FF` | 选中背景、浅色强调背景 |
| `--color-primary-subtle` | `#F6F7FF` | 很弱的蓝紫背景 |

主色使用规则：

- 一个页面最多出现 1 个主操作按钮。
- 不要大面积铺满蓝紫色。
- 蓝紫色主要用于「当前状态」和「主要动作」。
- 热力图和分类色可以包含蓝紫，但饱和度必须降低。

### 2.3 分类辅助色

分类颜色只用于小面积 icon chip、图表分布和热力图局部，不作为页面主色。

| 分类 | 背景 | 图标 / 数据色 | 用途 |
|---|---:|---:|---|
| 基本功 | `#DDEBFF` | `#4C7FE8` | 基础练习 |
| 手脚协调 | `#E8E4FF` | `#6B5BFF` | 协调类练习 |
| 曲目练习 | `#FFE5D6` | `#F26F45` | 曲目 / 歌曲练习 |
| 自由练习 | `#FFF1C8` | `#D69A00` | 未归类或自由练习 |
| 默认分类 | `#EEF0F3` | `#7B8492` | 新建分类默认色 |

### 2.4 状态颜色

| 状态 | 色值 | 用途 |
|---|---:|---|
| 成功 | `#22C55E` | 保存成功提示，不用于大面积 UI |
| 警告 | `#F59E0B` | 未保存、异常提醒 |
| 危险 | `#EF4444` | 删除、放弃计时确认 |
| 遮罩 | `rgba(23, 23, 23, 0.42)` | 保存弹窗背景遮罩 |

## 4. 布局与间距

### 4.1 画布

- 设计基准宽度：390px
- App 背景：`--color-bg`
- 页面左右边距：20px
- 顶部安全区：系统状态栏下 20–28px
- 底部为浮动胶囊导航预留：96–112px

### 4.2 间距 Scale

| Token | 值 | 用途 |
|---|---:|---|
| `--space-1` | 4px | 极小间距 |
| `--space-2` | 8px | 图标与文字 |

| `--space-4` | 16px | 卡片内部基础 padding |
| `--space-5` | 20px | 页面边距 |
| `--space-6` | 24px | 模块间距 |
| `--space-8` | 32px | 大区块间距 |

### 4.3 圆角

| Token | 值 | 用途 |
|---|---:|---|
| `--radius-sm` | 10px | 小 chip、输入框 |
| `--radius-md` | 16px | 小卡片 |
| `--radius-lg` | 22px | 主卡片 |
| `--radius-xl` | 28px | 底部弹窗、底部导航 |
| `--radius-full` | 999px | 胶囊按钮、圆形按钮 |

### 4.4 阴影

默认卡片阴影要非常轻：

```css
box-shadow: 0 12px 32px rgba(15, 23, 42, 0.05);
```

```css
box-shadow: 0 -16px 40px rgba(15, 23, 42, 0.10);
```

避免厚重投影、发光、强玻璃拟态。

## 6. 组件规范

### 6.1 主按钮 Primary Button

用途：页面最主要动作，如「开始计时」「保存记录」。

视觉：

- 高度：48–52px
- 圆角：14–16px 或 full pill
- 背景：`--color-primary`
- 文本：白色，15–16px，600
- 阴影：轻微蓝紫阴影，可选

状态：

| 状态 | 样式 |
|---|---|
| Default | `#5B63F6` |
| Pressed | `#4E56E8`，按钮缩放到 `0.98` |
| Disabled | 背景 `#DADDE8`，文字 `#FFFFFF` 70% |
| Loading | 保持按钮宽高，显示 spinner，禁止重复点击 |

功能说明：

- 「开始计时」：进入专注计时页，并启动计时。
- 「保存记录」：将本次练习时长、分类、备注保存为一条 PracticeSession，保存后返回「今天」页并刷新统计。

### 6.2 次级按钮 Secondary Button

用途：弱操作，如「放弃」。

视觉：

- 文本按钮或浅灰背景按钮
- 文本色：`--color-text-secondary`
- 高度：44px 以上

功能说明：

- 「放弃」：关闭保存弹窗并丢弃本次未保存计时。若已有有效时长，应弹出确认。

### 6.3 圆形控制按钮

用于计时页底部控制区。

| 按钮 | 尺寸 | 颜色 | 功能 |
|---|---:|---|---|
| 暂停 / 继续 | 72×72px | `--color-primary` | 暂停当前计时；暂停后点击继续 |
| 结束 | 52×52px | 白色卡片 / 浅灰 | 停止计时，打开保存弹窗 |
| 重置 | 52×52px | 白色卡片 / 浅灰 | 将当前未保存计时归零，需确认 |

计时中主按钮显示暂停 icon；暂停后显示播放 icon。

### 6.4 Icon Button

用于搜索、设置、添加、更多。

视觉：

- 触控区域：44×44px
- 图标：20–22px
- 默认无背景
- Pressed：浅灰圆形背景 `#F0F1F4`

功能说明：

| 按钮 | 位置 | 功能 |
|---|---|---|
| 搜索 | 今天页顶部 | 第一阶段可预留；点击后搜索分类或记录 |
| 设置 | 今天页顶部 | 第一阶段可预留；进入偏好设置 |
| 新增 | 今天页 / 分类页顶部 | 新增分类；或未来新增快捷练习 |
| 更多 | 计时页顶部 | 第一阶段可预留；未来放声音、屏幕常亮等 |
| 返回 | 子页面顶部 | 回到上一页，保留当前状态 |

### 6.5 Stat Card 数据卡片

用于今日总计、记录次数、总练习时长、本周时长等。

视觉：

- 背景：白色
- 圆角：16px
- Padding：14–16px
- 边框：`1px solid #ECEEF2`
- 数字：20–24px，700
- 标签：12–13px，次级文字

功能：

- 只展示关键数据，不可点击，除非未来有详情页。

### 6.6 Practice Hero Card 今日主卡片

用于「今天」页显示当前练习总时长和开始计时。

内容：

- 当前已练习总分钟数
- 辅助文字「当前已练习」
- 轻微节奏纹理
- 主按钮「开始计时」

规则：

- 这是首页视觉中心。
- 节奏纹理透明度要低，不影响可读性。
- 不放练习分类选择，分类绑定在保存弹窗完成。

### 6.7 Record Row 今日记录行

显示当天已保存练习记录。

内容：

- 分类 icon chip
- 分类名称
- 开始时间或备注摘要
- 时长
- 右侧 chevron

功能：

- 点击后可进入记录详情或编辑记录；第一阶段可先做为只读，保留 chevron 作为未来入口。

视觉：

- 高度：56–64px
- 圆角：14–16px
- 背景：白色
- 行间距：10–12px

用于保存弹窗和分类页筛选。

视觉：

- 圆角：999px

功能：

### 6.9 Category Card 分类卡片

用于「分类」页。

内容：

- 分类 icon chip
- 分类名称

- 右侧 chevron

功能：

- 点击进入分类详情或编辑分类。

### 6.10 Bottom Sheet 底部弹窗

用于「保存本次练习」。

视觉：

- 从底部进入
- 圆角：顶部 28px
- 背景：白色
- 遮罩：`rgba(23, 23, 23, 0.42)`
- 顶部 drag handle：36×4px，浅灰

交互：

- 点击「结束」后打开。
- 弹窗内选择分类、填写备注。
- 点击保存后关闭，返回今天页。
- 如果点击遮罩或下滑关闭，且有未保存时长，需要二次确认。

## 7. 页面规范与按钮功能

### 7.1 今天页

页面目标：让用户看到今天练了多久，并快速开始计时。

核心模块：

1. 顶部栏：`SnareLab` + 搜索 / 设置 / 新增
2. 日期：`7月9日 周四`
3. 今日摘要：今日总计、记录次数
4. 主卡片：当前已练习时长 + 开始计时
5. 今日记录列表
6. 底部导航

按钮功能：

| 按钮 | 功能 | 当前阶段实现建议 |
|---|---|---|
| 搜索 | 搜索分类或记录 | 可先隐藏或弹出“即将支持” |
| 设置 | 进入设置 | 可先隐藏或预留 |
| 新增 | 新增分类或快捷记录 | 建议进入新增分类 |
| 开始计时 | 进入专注计时并开始计时 | 必做 |
| 今日记录行 | 查看或编辑单条记录 | 可先只读，后续实现 |
| 底部「分类」 | 切换到分类页 | 必做 |
| 底部「统计」 | 切换到统计页 | 必做 |

### 7.2 专注计时页

页面目标：只记录时间，降低练习时干扰。

必须遵守：

- 不显示当前练习内容
- 不显示分类
- 不显示备注
- 不显示统计信息

核心模块：

1. 顶部返回 / 标题 / 更多
2. 圆形计时器
3. 状态文字：练习中 / 已暂停
4. 底部控制：结束 / 暂停继续 / 重置

按钮功能：

| 按钮 | 功能 |
|---|---|
| 返回 | 若计时中，提示是否结束或继续；若未开始则返回今天 |
| 更多 | 预留计时设置，如屏幕常亮、声音提示 |
| 结束 | 停止计时，打开保存本次练习弹窗 |
| 暂停 | 暂停计时，状态变为「已暂停」 |
| 继续 | 从暂停状态恢复计时 |
| 重置 | 清零当前未保存计时；若已有时长，需确认 |

### 7.3 保存本次练习弹窗

页面目标：在练习结束后，把时间绑定到分类。

数据绑定规则：

```text
计时时长 + 选择分类 + 可选备注 → 保存为一条练习记录
```

不要在计时开始前强制选择分类。

按钮功能：

| 按钮 / 控件 | 功能 |
|---|---|
| 分类 Chip | 选择本次练习归属分类，必选 |
| 自定义+ | 创建新分类并选中 |
| 备注输入框 | 记录本次练习内容，可选 |
| 保存记录 | 保存 PracticeSession，刷新今天页和统计 |
| 放弃 | 丢弃本次计时；需二次确认 |
| 遮罩 / 下滑 | 关闭弹窗；有未保存时长时需确认 |

表单规则：

- 分类必选。
- 备注非必填。
- 时长小于 10 秒时，保存按钮可禁用或提示「练习时间太短」。

### 7.4 分类页

页面目标：管理自定义练习分类，查看每类累计时长。

核心模块：

1. 页面标题：分类
2. 新增按钮
3. 筛选 Chip：全部 / 基本功 / 协调 / 曲目
4. 分类列表
5. 底部导航

按钮功能：

| 按钮 | 功能 |
|---|---|
| 新增分类 | 打开新增分类表单 |
| 筛选 Chip | 筛选分类列表 |
| 分类卡片 | 进入分类详情或编辑分类 |
| Chevron | 与点击卡片相同 |

分类字段建议：

```ts
type PracticeCategory = {
  id: string;
  name: string;
  color: string;
  icon: string;
  totalSeconds: number;
  lastPracticedAt?: string;
  createdAt: string;
  updatedAt: string;
};
```

### 7.5 统计页

页面目标：查看长期练习成果。

核心模块：

1. 四个统计卡片：总练习时长、本周时长、记录天数、记录次数
2. 周 / 月 / 年分段控制
3. 分类分布 Donut Chart
4. 练习热力图
5. 底部导航

按钮 / 控件功能：

| 控件 | 功能 |
|---|---|
| 周 / 月 / 年 | 切换统计时间范围 |
| Donut Chart 分类项 | 可高亮对应分类；第一阶段可只展示 |
| 热力图格子 | 点击显示当天练习总时长；第一阶段可只展示 |

图表规则：

- 不使用柱状图作为第一阶段主要趋势图。
- 热力图用于展示练习频率和强度。
- 热力图颜色从浅灰 / 浅蓝紫到主蓝紫，不要过饱和。
- Donut Chart 分类不超过 5 类；超过 5 类时合并为「其他」。

## 8. 交互与动效

动效原则：轻、快、有因果。

推荐：

- 页面切换：150–220ms fade / slide
- 底部弹窗进入：220–280ms，从底部上滑
- 按钮 pressed：缩放到 `0.98`，100–150ms
- 圆环进度：计时中平滑推进，但不要过度动画

必须支持：

- `prefers-reduced-motion`
- 触控区域不小于 44×44px
- icon-only 按钮需要 aria-label
- 表单错误显示在对应字段附近

## 9. 可访问性规范

最低要求：

- 正文与背景对比度 ≥ 4.5:1
- 大字号数字对比度 ≥ 3:1
- 所有按钮可键盘聚焦
- Focus ring 使用主色浅描边：

```css
outline: 2px solid rgba(91, 99, 246, 0.45);
outline-offset: 2px;
```

ARIA 建议：

| 元素 | 建议 |
|---|---|
| 开始计时 | `aria-label="开始计时"` |
| 暂停 | `aria-label="暂停计时"` |
| 继续 | `aria-label="继续计时"` |
| 结束 | `aria-label="结束计时并保存"` |
| 重置 | `aria-label="重置当前计时"` |
| 热力图 | 提供文本摘要，如「本月共练习 12 天，总时长 4 小时 10 分」 |

## 10. CSS Token 草案

```css
:root {
```

## 11. 第一阶段不做的内容

为了保持 MVP 清晰，以下内容暂不进入第一阶段：

- 架子鼓编谱
- 复杂练习库
- 节拍器
- BPM 记录
- 练习计划完成度
- 音频 / 视频上传
- 社区分享
- AI 练习分析
- 多设备云同步

这些功能以后可以加入，但不应影响当前 UI 的简洁度。
