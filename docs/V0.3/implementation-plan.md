# SnareLab Practice Log v0.3 实施计划

> **执行方式：** 严格一次只实施一个任务。每个任务完成后先提供测试范围，由车老板手动验证；在收到“验证通过”前，不得开始下一任务。验证通过后，立即更新 `memory-bank/process.md` 与 `memory-bank/arch.md`。

**目标：** 在现有 V0.2 本地优先 PWA 基础上，完成与已确认 V0.3 原型一致的三栏练习记录体验：今日、记录、统计，并支持图片附件、设置内的分类/标签管理，以及月/年统计。

**实施策略：** 不重建项目。保留 React、TypeScript、Zustand、Dexie、IndexedDB、Vitest、Testing Library 和 Playwright；在现有 V0.2 领域模型、仓储和组件之上进行可迁移升级。先稳定数据与路由，再完成计时保存、记录编辑、统计，最后做视觉一致性和 PWA 验收。

## 全局约束

- 每次写代码前完整阅读 `docs/V0.2/`、`docs/V0.3/`、`memory-bank/` 与 V0.3 交互原型。
- 原型是 V0.3 UI 与交互的唯一视觉基准；所有可见文本使用中文，只有 `SnareLab`、`Today`、`Jul 12` 例外。
- 仅保留“今日 / 记录 / 统计”三个主导航入口；分类与标签管理迁移到设置内。
- 练习时长、开始时间和结束时间在保存后不可编辑；删除记录为硬删除。
- 图片仅限本地 IndexedDB：最多 6 张，每张压缩后不超过 2MB；不引入云端、账户或音视频。
- 统计月/年切换必须同时影响热力图、分类统计和标签统计。
- 每个任务至少运行相关单元测试、`npm run typecheck`、`npm run build`；涉及页面交互时额外运行 Playwright 检查。

## 目标文件结构

- `src/types/`：V0.3 图片附件、统计和界面状态的共享类型。
- `src/database/`：Dexie V0.3 schema、迁移和默认数据。
- `src/repositories/`：记录、分类、标签和图片附件的持久化边界。
- `src/services/`：图片压缩、统计聚合、草稿恢复和日期分组逻辑。
- `src/components/`：热力图、统计图、Bottom Sheet、图片网格、设置入口和底部导航。
- `src/pages/Today/`、`Timer/`、`Log/`、`Statistics/`：三条主流程及沉浸式计时页。
- `tests/` 与同目录 `*.test.tsx`：单元、组件、集成和浏览器回归测试。

## 执行规则

1. 每次开始一个任务前，先用计划模式复读该任务的目标、影响文件、验证步骤和不在本任务范围内的事项。

5. 每个任务完成后都回复：`车老板！已完成！`
