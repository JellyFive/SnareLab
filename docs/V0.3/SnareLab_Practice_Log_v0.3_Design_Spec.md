# SnareLab Practice Log v0.3 设计规范

**版本：** v0.3  
**状态：** 已确认  
**产品：** SnareLab Practice Log  
**语言：** 除 `SnareLab` 标识与 Today 页标题/英文短日期（`Today`、`Jul 12`）外全部使用中文

## 1. 设计原则

- 练习优先：开始练习始终是今日页最明显的动作。
- 信息诚实：时长、日期和统计口径必须清楚，不用装饰掩盖数据含义。
- 管理后置：分类与标签存在，但不占据主导航。
- 冷静有力：以浅色界面承载高频操作，只让靛蓝强调当前状态与主操作。
- 移动优先：以 390px 宽度为基准，所有可点击控件至少 44px。

## 2. 视觉方向

界面像一块安静、精确的练习控制台：中性浅灰背景、白色表面、克制边界与清晰数字。圆形计时器和两张圆形统计图是唯一的强形状，用于表达时间流逝与练习构成。

不使用渐变文字、装饰性光晕、营销式大标题或与任务无关的卡片堆叠。

## 3. 设计令牌

### 3.1 颜色

| 令牌 | 值 | 用途 |
| --- | --- | --- |
| 背景 | `#F5F6F8` | 页面底色 |
| 表面 | `#FFFFFF` | 内容区、底部栏、Bottom Sheet |
| 主文字 | `#1C1D20` | 标题与核心数据 |
| 次文字 | `#626875` | 说明与次要信息 |
| 边界 | `#E4E7EC` | 分隔与输入边界 |
| 主色 | `#535BF2` | 主按钮、选中状态、热力图 |
| 主色浅层 | `#EEF0FF` | 选中背景 |
| 成功 | `#199C63` | 已保存反馈 |
| 提醒 | `#C98512` | 草稿与待处理状态 |
| 危险 | `#D83C3E` | 删除与不可逆操作 |

分类色用于分类徽标与统计图扇区；标签色比分类色更低饱和，不能压过主色和分类色。

### 3.2 字体与数字

```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", Arial, sans-serif;
font-variant-numeric: tabular-nums;
```

| 层级 | 字号 | 字重 | 用途 |
| --- | ---: | ---: | --- |
| 计时数字 | 60px | 700 | 计时页圆环中心 |
| 主要数字 | 32px | 700 | 今日时长、累计时长 |
| 页面标题 | 28px | 700 | 今天练了吗、记录、统计 |
| 模块标题 | 18px | 700 | 热力图、练习记录 |
| 正文 | 15px | 500 | 列表和表单 |
| 辅助文字 | 13px | 500 | 时间、说明、统计口径 |

### 3.3 间距、圆角与阴影

间距使用 4、8、12、16、20、24、32px。页面左右内边距 20px，底部导航预留 96px。

- 文字输入与小控件：10px 圆角。
- 内容面板：14px 圆角。
- Bottom Sheet 顶角：20px。
- 仅 Bottom Sheet 可使用向上阴影；普通内容以边界或浅底色区分，不叠加宽大阴影。

## 4. 组件规范

### 4.1 按钮

| 类型 | 视觉 | 使用场景 |
| --- | --- | --- |
| 主按钮 | 靛蓝实底、白字、48px 高 | 开始练习、保存记录 |
| 次按钮 | 白底、1px 边界、深色字、44px 高 | 暂停、重置、筛选、取消 |
| 危险按钮 | 红色文字或红色实底 | 删除、确认放弃 |
| 图标按钮 | 44px 方形点击区 | 返回、设置、筛选、关闭 |
| 圆形控制 | 56px/72px 圆形 | 计时页控制，仅限计时场景 |

所有按钮至少包含默认、悬停、按下、焦点、禁用状态。图标按钮必须有中文 `aria-label`。

### 4.2 分类与标签

- 分类使用单选下拉或选择列表，并显示色点与中文名称。
- 标签使用可多选胶囊控件；选中态为主色浅层与勾选符号。
- “新增分类”“新增标签”使用带加号的次级文字按钮，不与主保存按钮竞争。

### 4.3 Bottom Sheet

- 从底部进入，220ms 至 260ms，使用缓出曲线。
- 打开后焦点进入关闭按钮，Tab 键在面板内循环。
- 内容可滚动，底部保存操作保持可见。
- 删除确认单独展示明确后果，取消为更安全的默认动作。

### 4.4 图片附件

- 以 3 列正方形缩略图网格显示。
- 最后一个格子为“添加图片”图标按钮。
- 单张图片显示删除入口；达到 6 张后“添加图片”禁用并说明数量上限。
- 图片加载时显示固定尺寸骨架，不造成页面跳动。

## 5. 页面布局

### 5.1 今日

```text
SnareLab                                  设置
Today                                  Jul 12
今日练习时长              总练习时长
开始练习
2026 年 7 月              月切换
紧凑月度热力图            本月练习天数 / 累计时长
今日练习记录
记录行（不可点击）
今日 / 记录 / 统计
```

主按钮置于首屏。热力图用于当月回顾，不能抢占开始练习的视觉权重。
今日练习时长和总练习时长必须是两个独立的等宽面板，不共享外框或分隔线。
热力图标题不重复显示年月；年月由日历面板自身的“7 月”与月切换控件表达。
热力图右侧使用与主界面一致的紧凑月度摘要面板，显示练习天数和累计时长，不使用裸文本堆叠。

### 5.2 计时

```text
返回              SnareLab              更多
                圆形计时器
                  00:24:18
                   练习中
          重置       暂停       停止
```

圆环进度反映当前分钟内的秒数，不作为目标或完成率。暂停状态将中心文字改为“已暂停”，主控制改为“继续”。

### 5.3 保存 Bottom Sheet

先显示只读时长，随后是分类、标签、笔记和图片。主保存按钮置底；分类未选中时按钮禁用。

### 5.4 记录

```text
SnareLab                 记录             筛选
筛选摘要（存在筛选时显示）
周六   ●  2026 年 7 月 12 日
09:10  ○  紧凑记录行
19:30  ○  紧凑记录行
周五   ●  2026 年 7 月 11 日
20:05  ○  紧凑记录行
今日 / 记录 / 统计
```

记录行可点击。左侧使用细竖线串联日期实心节点与记录空心节点，日期与开始时间帮助快速扫描；卡片正文维持紧凑的小字号密度。图片以“图片 3 张”或首图缩略图提示，不铺满记录列表。

### 5.5 统计

```text
SnareLab                 统计             设置
概览 / 分类 / 标签
概览：年度热力图、年度摘要、月份总结列表
月份：完整月视图热力图、月度摘要、分类分布、近期记录
日期：当天时长、次数、分类分布与记录详情入口
分类：分类时长、月度趋势、分类明细
标签：标签时长排行与组合分析
今日 / 记录 / 统计
```

概览页必须使用由 12 个紧凑月历组成的全年热力图，并提供从少到多的强度图例。月份总结行是明确的进入月度详情入口。月度详情的热力图必须展示完整自然月，包含月份、前后月切换、星期标题和全部日期；日期单元是进入日详情的入口。

分类图和标签图使用不同、可区分的靛蓝体系辅助色。圆图置于统计卡上方居中，中心显示汇总指标；下方每个条目同时给出色点、名称、时长、百分比和与扇区对应颜色的横向占比条。标签数据按练习时长表达，组合分析使用紧凑卡片，不引入绿色主题或原型中的额外训练字段。

### 5.6 设置

设置项为“分类管理”“标签管理”“页面主题”。前两项可进入管理界面；页面主题为禁用行并标记“即将开放”。

## 6. 响应式与无障碍

- 320px 至 480px：单列移动布局，统计摘要保持三等宽但文字可换行。
- 481px 以上：内容最大宽度 720px，保持移动应用的阅读密度。
- 正文与背景对比度至少 4.5:1；大数字至少 3:1。
- 支持键盘焦点、屏幕阅读器名称与 `prefers-reduced-motion`。
- 禁用和空状态使用文字解释，不仅依赖颜色。

## 7. 文案规则

- 页面和按钮使用简洁中文动词，例如“开始练习”“保存记录”“继续保存”。
- `SnareLab` 是唯一保留的英文品牌标识。
- 不使用“提交”“操作成功”等系统化空泛文案；反馈要说明具体结果，例如“已保存本次练习记录”。
