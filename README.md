SnareLab is a local-first personal drum practice system focused on snare rhythm training for the V1.0 MVP.

## Project Structure

| Directory | Purpose |
|---|---|
| `requirements/` | Product requirements, rhythm catalog requirements, and source-of-truth product scope. |
| `design/` | Engineering design documents and implementation-oriented architecture decisions. |
| `reviews/` | Review documents, product/design critiques, and pre-development sign-off notes. |
| `frontend/` | Frontend application workspace, UI prototype references, and frontend-owned catalog source files. |
| `backend/` | Backend workspace placeholder. V1.0 is local-first and does not require a backend by default. |
| `tests/` | Test strategy, acceptance scenarios, fixtures, and future automated test suites. |

## Documentation Rule

Every Markdown document in this project must keep a `变更记录` section near the top. When a document changes, add a new row describing the date, version or scope, and what changed.
